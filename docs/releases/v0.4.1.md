## rookery v0.4.1

v0.4.1 turns Slack-origin masters into much better workspace citizens: they can safely gather read-only context beyond the current thread, understand rich Slack messages and attachments, and identify the people, bots, and channel behind each turn.

_19 commits since v0.4.0._

### 💬 Read-only Slack workspace tools

- **Six focused Slack tools.** Slack-origin masters can now use `read_thread`, `read_channel`, `list_channels`, `get_user_info`, `get_permalink`, and `download_file` from both Claude and Codex sessions.
- **Context beyond the current thread.** Read recent messages from another channel the bot belongs to, inspect a specific thread, resolve `#channel` names and user IDs, and generate durable links to Slack messages.
- **Graceful permission guidance.** Missing scopes, disconnected Slack state, or channels the bot has not joined return actionable tool errors without terminating the master turn.
- **No Slack write surface.** The new capability remains read-only from Slack's perspective; it does not add message posting, reactions, uploads, or other outbound actions.

### 📎 Full-fidelity transcripts and attachments

- **Block Kit and attachment text is preserved.** Slack reads now melt message text, blocks, rich text, and legacy attachments into one transcript, so CI cards and integration messages no longer appear empty.
- **Files stay visible.** Image and file-only messages render `[file: … id=…]` markers instead of disappearing from the transcript.
- **On-demand file inspection.** `download_file` fetches only the selected attachment into rookery's contained Slack file directory, after which the master can inspect it locally. Attachments are not downloaded automatically.

### 🧭 Clear sender, channel, and author identity

- **Every Slack turn carries its origin.** A compact header supplies the current sender, channel, and thread timestamp, with best-effort name resolution and bounded one-line sanitization.
- **`masterName` reaches Slack surfaces.** The configured master name now appears in assistant thread titles, greetings, and the master's own transcript bot label.
- **Real transcript author labels.** Human authors resolve to names where possible, third-party bots retain their own bot name, and only rookery's bot uses the configured master name.

### 🔐 Slack app prerequisites

- The conversational Slack path still requires the `message.channels` event, `channels:history`, and membership in target channels.
- Cross-channel reads and name resolution additionally use `channels:read`, `groups:read`, `groups:history`, and `users:read`; attachment download uses `files:read`. Missing scopes degrade per tool instead of breaking the session.

### 📦 Notes

- macOS builds are signed and notarized. Linux AppImage and unsigned Windows installer builds are included in the release assets.
- Updater metadata and stable latest-download filenames are published for all supported desktop platforms.
