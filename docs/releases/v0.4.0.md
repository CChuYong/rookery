## rookery v0.4.0

v0.4.0 makes managed capabilities a first-class part of rookery, adds a clear view into Claude's dynamic workflow activity, and hardens the desktop experience around prompt composition, dialogs, and worker lifecycle changes.

_92 commits since v0.3.1._

### 🧩 Capability Center and managed capabilities

- **A reviewable capability catalog.** Import skills, register MCP servers, and assemble reusable multi-server MCP packs from the Capability Center.
- **Scoped defaults and effective previews.** Apply capabilities rookery-wide or per repository, then inspect the effective provider/agent configuration before it reaches a session or worker.
- **Provider-native application.** Trusted packs compile into the native Claude and Codex runtime surfaces while keeping generated state isolated from the user's own provider configuration.
- **Repository sharing and live reload.** Share repository-local capability packs with strict validation and trust invalidation, watch them for changes, and reload live workers immediately or when they next become idle.
- **Contextual capability commands.** Slash-command suggestions now reflect the selected session or worker, with direct `/skills`, `/hooks`, and `/mcp` navigation into Capability Center.
- **Safer MCP configuration.** Public server settings stay reviewable, secrets remain write-only, invalid changes roll back cleanly, and generated runtime files are contained and garbage-collected.

### ⚙️ Claude Dynamic Workflow activity

- **One Activity sidebar.** Claude Dynamic Workflow phases and native nested agents now appear in a unified activity surface without being confused with rookery workers.
- **Live progress with bounded detail.** Phase status updates arrive in real time, transcripts load on demand, and reconnects restore the current workflow view.
- **A deliberate privacy boundary.** The desktop receives allowlisted summaries instead of raw prompts, scripts, paths, results, logs, or previews, and workflow activity never drives Worker state.

### ✍️ More reliable prompts and dialogs

- **A Lexical-based prompt editor.** The composer now uses a real editor state model while preserving file chips, mentions, slash actions, Markdown shortcuts, drafts, attachments, and Side conversations.
- **Correct paste undo with Korean IME.** Undoing a paste no longer corrupts the next Hangul composition or drops its initial consonant.
- **Protected drafts.** Prompt-bearing dialogs ignore accidental Escape and outside-click dismissal, while explicit cancel actions still work. Worker Spawn also stays open until the daemon confirms success and preserves its draft after failures.
- **Unclipped menus and resilient forms.** The “Open in” menu renders outside the header's clipping boundary, and Skill Import keeps labels, controls, and actions usable at constrained sizes.

### 🛡 Fleet and navigation hardening

- **Monotonic worker deletion.** In-flight deletes are daemon-owned and idempotent; late events, stale snapshots, and reconnects can no longer resurrect a removed worker in the desktop.
- **Persistent repository navigation.** Repository group folding and archive visibility survive restarts, with stale persisted keys cleaned up automatically.
- **A quieter attention queue.** Orphaned workers no longer appear in “Needs you now,” keeping the list focused on actionable items.

### 📦 Notes

- Capability packs remain disabled until they have been explicitly reviewed and trusted; sensitive MCP values are never returned through applied-state, preview, or invocation payloads.
- macOS builds are signed and notarized. Linux AppImage and Windows installer builds are also included in the release assets.
- The desktop bundle continues to ship its required Node.js 22 runtime and updater metadata for supported platforms.
