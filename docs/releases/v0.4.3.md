## rookery v0.4.3

v0.4.3 is a follow-up stability release to the v0.4.2 `background`-state work. An audit of that change surfaced four more places where a daemon-side state or field wasn't fully reflected in the desktop UI — this release closes them.

_5 commits since v0.4.2._

### 🏷️ Live state now shows up right away

- **Codex workers show their badge immediately.** A worker running on the Codex backend used to render with no "Codex" badge for its entire run — identical to a Claude worker — and the badge only appeared after the worker finished or the app reconnected. The badge now shows the moment the worker is spawned, for master-, Slack-, and automation-spawned workers alike.
- **The model picker no longer drops to 3 defaults.** After the window sat open for a while, the chat model dropdown could silently collapse to the three built-in defaults (Opus 4.8 / Sonnet 4.6 / Haiku 4.5), and only a restart brought back the full model catalog. A reconnect (e.g. after the laptop wakes from sleep) that momentarily failed to refresh the catalog no longer overwrites it — the daemon now serves the last known-good catalog through the blip.

### 🎛️ Desktop UI polish

- **Worker-header controls stay reachable in the `background` state.** On a narrow worker pane, a background worker's wide status badge plus its activity chip could push the far-right panel-toggle buttons off the clipped header, making them unclickable — the only way back to a hidden panel in dock mode. The status badge and activity chip now collapse responsively so the panel controls always stay in reach.
- **Attention bell shows worker status in your language.** Worker-failure and review rows in the header attention bell printed the status word in raw English (e.g. `error`, `stopped`) even in the Korean UI. They now use the same translated status labels as the rest of the app.

### 📦 Notes

- macOS builds are signed and notarized. Linux AppImage and unsigned Windows installer builds are included in the release assets.
- Updater metadata and stable latest-download filenames are published for all supported desktop platforms.
