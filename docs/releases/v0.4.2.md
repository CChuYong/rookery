## rookery v0.4.2

v0.4.2 is a stability release that makes the worker `background` state a first-class, interactive state everywhere in the desktop app, and cleans up how abnormal turn terminations are surfaced.

_12 commits since v0.4.1._

### 🟢 `background` is now a fully live worker state

A worker enters `background` when its turn has ended but harness-tracked background tasks (e.g. `run_in_background` shells) are still running. Several desktop surfaces treated it as "not live" and quietly locked up. This release closes those gaps so a background worker behaves exactly like a running one until its work is truly done.

- **Composer stays writable.** You can queue follow-up messages to a worker while its background tasks run, instead of the input being disabled.
- **Stop and the live filter keep working.** The Stop control and the "live" fleet filter now include background workers.
- **Optimistic and queued bubbles are retained.** Messages you send during a `running → background` transition no longer vanish; the optimistic and queued bubbles persist and reconcile normally.
- **Status visuals reflect a live state.** Status badges and status colors render `background` as an active state.

### 🔔 Truthful attention and notifications

- **Notify on worker error, not only failed.** Terminal `error` transitions now raise attention alongside `failed`.
- **Unread on a stopped settle.** A worker that settles on `stopped` correctly marks the session unread so it surfaces in the attention queue.
- **Slack flags failed transitions.** The Slack activity relay flags genuinely failed worker transitions instead of relying on the retired `done` state.

### ⚠️ Cleaner abnormal-termination signals

- **`completed` is no longer shown as a warning.** The normal-finish sentinel from the Claude SDK is filtered at the source, so a healthy turn never surfaces a red `⚠ completed` in the metrics header.
- **Refreshed dead-turn taxonomy.** The "Turn ended abnormally" notice now matches the current SDK `TerminalReason` enum (`blocking_limit`, `rapid_refill_breaker`, `prompt_too_long`, `image_error`, `model_error`). Genuine failures are noticed again, while intentional interrupts (`aborted_streaming`/`aborted_tools`), `max_turns` (which has its own notice), and background/deferred/completed reasons are deliberately excluded.

### 📦 Notes

- macOS builds are signed and notarized. Linux AppImage and unsigned Windows installer builds are included in the release assets.
- Updater metadata and stable latest-download filenames are published for all supported desktop platforms.
