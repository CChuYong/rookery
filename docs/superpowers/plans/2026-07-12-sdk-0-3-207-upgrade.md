# SDK 0.3.207 Upgrade + Feature Adoption Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bump `@anthropic-ai/claude-agent-sdk` 0.3.195 → 0.3.207 and adopt terminal_reason consumption, level-based background-task tracking, interrupt receipts; investigate (and likely close as not-adopted) depth-2+ nested trees.

**Architecture:** All SDK wire-shape knowledge stays in the adapter (`src/core/claude-backend.ts` `translate()`), which emits provider-neutral `AgentEvent`s consumed by `Worker`/`MasterAgent`. New SDK signals therefore land as: adapter decode → new/extended `AgentEvent` member → worker consumption → (where user-visible) `WorkerEventData`/desktop rendering. Spec: `docs/superpowers/specs/2026-07-12-sdk-0-3-207-upgrade-design.md`.

**Tech Stack:** TypeScript (NodeNext ESM), vitest, better-sqlite3, Electron/React (desktop), Claude Agent SDK 0.3.207.

## Global Constraints

- **Node 22 mandatory** for every npm/node command (`nvm use 22` first; better-sqlite3 is ABI 127).
- ESM NodeNext: relative imports need `.js`; type-only imports need `import type` (`verbatimModuleSyntax`).
- Code comments in **English**. Chat with the user is Korean; code is not.
- Gates per task: `npm run typecheck && npm test` (root). Tasks touching desktop also: `npm -w apps/desktop run typecheck && npm -w apps/desktop test`.
- Desktop i18n: every new user-facing renderer string gets ko+en catalog entries (same key set — parity test enforces).
- Worker notices are plain English text (`{kind:"notice", text}`) — do NOT add daemon `notice.*` i18n codes for worker-level notices (that machinery is master-only; spec Phase 2-A is amended accordingly).
- No changes to `*_TOOL_NAMES` constants are needed anywhere in this plan (no tool names change).
- Commit trailer: `Co-Authored-By: Claude Opus 4.8 (1M context) <noreply@anthropic.com>`.
- Branch: `sdk-0.3.207` (worktree), FF-merged to main at the end.
- SDK type reference: the 0.3.207 `sdk.d.ts` (in `node_modules/@anthropic-ai/claude-agent-sdk/` after Task 1). Key shapes, verified against the published tarball:
  - `SDKBackgroundTasksChangedMessage = { type:'system', subtype:'background_tasks_changed', tasks: {task_id, task_type, description}[], uuid, session_id }` — REPLACE semantics; SDK docs: do NOT correlate with task_* edge frames.
